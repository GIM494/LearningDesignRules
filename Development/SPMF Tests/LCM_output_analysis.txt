GENERAL COMMENTS:\
So for all of these, the main take away seems to be that the usefulness and interestingness of an FI relies on the presence of at least one \'93critical\'94 attribute that is specific to a coherently related subset of classes. Examples of critical attributes are \'93extends X\'94, \'93has annotation Y\'94, or \'93has function Z\'94. For example, take the following attributes:\
\pard\tx720\tx1440\tx2160\tx2880\tx3600\tx4320\tx5040\tx5760\tx6480\tx7200\tx7920\tx8640\pardirnatural\partightenfactor0

2 class defines constructor with no parameters\
4 class has member field of type String\

7 is public class\
58 class has member field type of type String\
And consider also the following FIs:\
     (A)  1 2 4 7\
     (B) 1 2 4 6 7 \
     (C) 1 2 4 6 7 58 \
     (D) 1 2 4 7 58\
The first FI (A) is not very interesting or useful because all the features are very general and could apply to any number of classes that are completely unrelated. However, the second FI (B) is significantly more interesting simply because of the presence of attribute 6 since it restricts the application of the broader characteristics around it to a set of coherently related classes (i.e., the classes that all extend the DTO class); attribute 6 is a \'93critical\'94 attribute because without it the FI would be too general.\
\
We have discussed possibly trying to make some sort of interestingness measure or specifying a few attributes that are particularly interesting. We might be able to use this concept of a critical attribute when trying to quantify interestingness or specifying more specific attributes to highlight more interesting rules that are produced.\
